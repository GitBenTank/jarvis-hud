## ADR status

In this repository, **Accepted** means the decision is the **recorded direction** in version control—the same convention as ADR-0001 through ADR-0006.

If you adopt **Proposed** for decisions that await explicit human sign-off before they are treated as binding, use that label **consistently** for all ADRs in that state, and move to **Accepted** only after sign-off. Do not mix meanings across files.
