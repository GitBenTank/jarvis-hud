# Investor read pack

**Purpose:** One **fixed** reading order for advisors and investors—about **15 minutes** total. Use the same four links every time; consistency builds trust.

**Shortcut in the HUD:** [/docs/tati](/docs/tati) (same content, minimal layout).

---

## Read in this order

1. **[Gener8tor pitch](./gener8tor-pitch.md)** — Story + demo framing (consequence before chrome).
2. **[Room playbook](./room-playbook-v1.md)** — How we show up in the room (restraint, objections in Q&A).
3. **[Thesis Lock (ADR)](../decisions/0001-thesis-lock.md)** — Constitutional rule: propose vs approve vs execute vs receipts (single source of truth; do not swap for another doc).
4. **[Flagship team bundle](./flagship-team-bundle-v1.md)** — Agent layer as a **system** (Alfred + Research + Creative), not one magic assistant.

---

## What not to send in the first pass

- The full `/docs` library or `?library=all`
- Integration roadmaps (e.g. operator integration phases) unless they ask for engineering depth
- Setup, env, and runbooks unless they are installing the stack
- Architecture dumps unless diligence requests them

**Live meeting:** Open with the problem and **demo** (`/demo`); use docs when they ask.
